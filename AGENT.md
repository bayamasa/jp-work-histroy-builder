# AGENT.md

## Git 運用ルール

mainブランチで作業中にファイル差分が発生した場合、以下の手順で進めること：

1. **ブランチ作成**: `feature/xxx` や `fix/xxx` など適切な名前でブランチを切る
2. **コミット**: 変更内容をコミットする
3. **プッシュ**: リモートにプッシュする
4. **PR作成**: GitHub上でPRを作成する
5. **マージ**: PRをマージする
6. **mainを最新化**: ローカルのmainブランチに戻り `git pull` で最新化する

直接mainブランチにコミット・プッシュしないこと。

## 機密ファイルの取り扱い

- `.personal/credential.yaml`の対話形式での聞き取り、ファイルの読み取り、存在チェックは **絶対に行わない**
- これらのファイルには個人を特定できる情報（氏名・住所・電話番号等）が含まれる
